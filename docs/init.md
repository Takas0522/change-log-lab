# ToDoアプリケーションの構築

## 概要

このドキュメントでは、シンプルかつ実用的な ToDoアプリケーションの設計・実装方針について説明します。

---

## 機能要件

### 基本機能

| 機能 | 説明 |
|------|------|
| タスク作成 | タイトル・説明・期限を指定してタスクを追加できる |
| タスク一覧表示 | 登録済みのタスクを一覧で確認できる |
| タスク編集 | タスクの内容を更新できる |
| タスク削除 | 不要なタスクを削除できる |
| 完了チェック | タスクを完了済みとしてマークできる |

### 追加機能

- タスクのフィルタリング（全件 / 未完了 / 完了済み）
- タスクの優先度設定（高 / 中 / 低）
- 期限切れタスクのハイライト表示

---

## 技術スタック

```
WebApp : ASP.NET WebForms(.NET Framework)
Windows App: Windows Forms(.NET Framework)
データベース   : SQLite
```

## 開発の進め方

1. **開発環境の確認** — 現在動いているOSを確認しそのOSの名前を高らかに宣言する
2. **要件定義** — ユーザーストーリーの作成と機能要件の明確化
  - `docs`に格納。長くなるためセクションごとにファイル分割すること
3. **開発環境のベース構築** — src内が空であるため、プロジェクトの作成と依存パッケージのインストール
4. **環境構築** — 依存パッケージのインストール・DB初期化
5. **実装** — 機能ごとに実装を進める
6. **リリース** — ビルド
7. **動作確認とキャプチャの取得** — 動作確認とスクリーンショットの取得
