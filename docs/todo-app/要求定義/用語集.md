# 用語集（Glossary）

**文書ID**: SRS-001-GLOSSARY  
**バージョン**: 1.0  
**作成日**: 2024-01-15  
**関連文書**: SRS-001-todo-app.md

## 1. はじめに

本文書は、高機能ToDoアプリケーションプロジェクトで使用される用語、略語、定義を明確にするものである。ステークホルダー間の共通理解を確立し、誤解やコミュニケーションエラーを防ぐことを目的とする。

## 2. 用語定義

### 2.1 アプリケーション固有用語

| 用語 | 英語表記 | 定義 | 同義語 | 関連用語 |
|---|---|---|---|---|
| **ToDo** | To-Do | タスク管理の基本単位となる項目。タイトル、内容、投稿日時、ステータス、ラベルから構成される。 | タスク、アイテム | ToDo ID、タイトル、内容 |
| **ToDo ID** | To-Do ID | 各ToDoを一意に識別するための識別子。システムにより自動生成される。 | タスクID | ToDo |
| **タイトル** | Title | ToDoの簡潔な見出し。最大200文字。必須項目。 | 件名、見出し | ToDo |
| **内容** | Content, Description | ToDoの詳細説明。最大5000文字。任意項目。 | 詳細、説明、本文 | ToDo |
| **投稿日時** | Created At, Timestamp | ToDoが作成された日時。システムにより自動設定される。 | 作成日時、登録日時 | 更新日時 |
| **更新日時** | Updated At | ToDoが最後に編集された日時。編集時にシステムが自動更新する。 | 編集日時、変更日時 | 投稿日時 |
| **ステータス** | Status | ToDoの進捗状態を示す属性。4つの値（未着手、着手中、完了、放棄）のいずれかを取る。 | 状態、進捗状況 | ステータス管理 |
| **未着手** | Not Started, Todo | ToDoがまだ開始されていない状態。ToDo作成時の初期ステータス。 | 未開始、TODO | ステータス |
| **着手中** | In Progress, Doing | ToDoが現在作業中である状態。 | 進行中、作業中、DOING | ステータス |
| **完了** | Completed, Done | ToDoが完了した状態。 | 終了、DONE | ステータス |
| **放棄** | Abandoned, Canceled | ToDoを中止・放棄した状態。 | 中止、キャンセル | ステータス |
| **ラベル** | Label, Tag | ToDoを分類・整理するためのタグ。色とラベル名で構成される。1つのToDoに複数設定可能。 | タグ、カテゴリ | ラベルID、ラベル名、色 |
| **ラベルID** | Label ID | 各ラベルを一意に識別するための識別子。システムにより自動生成される。 | タグID | ラベル |

| **絞り込み** | Filter | 特定の条件に合致するToDoのみを表示する機能。 | フィルタ、検索 | 検索条件 |
| **検索** | Search | キーワードによりToDoを検索する機能。 | サーチ | 絞り込み |
| **一覧画面** | List View | ToDoを一覧形式で表示する画面。 | リスト画面、一覧表示 | 詳細画面 |
| **詳細画面** | Detail View | 個別ToDoの詳細情報を表示する画面。 | 詳細表示 | 一覧画面、編集画面 |
| **編集画面** | Edit View | ToDoの情報を編集する画面。 | 編集フォーム | 詳細画面 |

## 4. 要件ID命名規則

### 4.1 機能要件ID

**形式**: `REQ-FUNC-XXX`
- REQ: Requirement（要件）
- FUNC: Functional（機能）
- XXX: 3桁の連番（001〜999）

**例**: 
- REQ-FUNC-001: ToDo作成機能
- REQ-FUNC-013: タグによる絞り込み機能

### 4.2 非機能要件ID

**形式**: `REQ-[CATEGORY]-XXX`
- REQ: Requirement（要件）
- CATEGORY: カテゴリ（PERF/SEC/USE/REL/MAIN/COMP/SCALE）
- XXX: 3桁の連番（001〜999）

**カテゴリ一覧**:
- PERF: Performance（性能）
- SEC: Security（セキュリティ）
- USE: Usability（ユーザビリティ）
- REL: Reliability（信頼性）
- MAIN: Maintainability（保守性）
- COMP: Compatibility（互換性）
- SCALE: Scalability（スケーラビリティ）

**例**:
- REQ-PERF-001: ToDo一覧表示時間
- REQ-SEC-003: XSS対策

### 4.3 制約条件ID

**形式**: `CONST-[TYPE]-XXX`
- CONST: Constraint（制約）
- TYPE: 種別（TECH/BIZ/LAW/OPS）
- XXX: 3桁の連番（001〜999）

**種別一覧**:
- TECH: Technical（技術制約）
- BIZ: Business（ビジネス制約）
- LAW: Legal（法規制・コンプライアンス）
- OPS: Operations（運用制約）

**例**:
- CONST-TECH-001: Webアプリケーションとして実装
- CONST-LAW-001: 個人情報保護法に準拠

### 4.4 前提条件ID

**形式**: `ASSUME-[TYPE]-XXX`
- ASSUME: Assumption（前提）
- TYPE: 種別（USER/TECH/BIZ）
- XXX: 3桁の連番（001〜999）

**例**:
- ASSUME-USER-001: ユーザーは基本的なPC/スマートフォン操作ができる
- ASSUME-TECH-004: CI/CD環境が構築されている

### 4.5 テストケースID

**形式**: `TC-XXX-YYY`
- TC: Test Case（テストケース）
- XXX: 対応する機能要件番号（001〜999）
- YYY: テストケース番号（001〜999）

**例**:
- TC-001-001: ToDo作成_正常系_全項目入力
- TC-001-002: ToDo作成_異常系_タイトル未入力

## 9. エラーコード

### 9.1 HTTPステータスコード

| コード | 名称 | 意味 | 用途 |
|---|---|---|---|
| 200 | OK | 成功 | 正常な処理 |
| 201 | Created | 作成成功 | リソース作成 |
| 400 | Bad Request | 不正なリクエスト | バリデーションエラー |
| 401 | Unauthorized | 未認証 | 認証エラー |
| 403 | Forbidden | 禁止 | 認可エラー |
| 404 | Not Found | 見つからない | リソースが存在しない |
| 409 | Conflict | 競合 | データ競合 |
| 500 | Internal Server Error | サーバーエラー | システムエラー |
| 503 | Service Unavailable | サービス利用不可 | メンテナンス中 |

### 9.2 アプリケーションエラーコード

**形式**: `ERR-[CATEGORY]-XXX`

| カテゴリ | 説明 | 範囲 |
|---|---|---|
| VAL | Validation（バリデーション） | ERR-VAL-001～099 |
| AUTH | Authentication（認証） | ERR-AUTH-001～099 |
| AUTHZ | Authorization（認可） | ERR-AUTHZ-001～099 |
| DB | Database（データベース） | ERR-DB-001～099 |
| SYS | System（システム） | ERR-SYS-001～099 |

**例**:
- ERR-VAL-001: タイトルが未入力です
- ERR-AUTH-001: ログインが必要です
- ERR-DB-001: データの保存に失敗しました
