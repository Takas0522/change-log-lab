## 文書構成

### SDD-001: 高機能ToDoアプリケーション 詳細設計書

**ファイルパス**: `SDD-001-todo-app.md`

**対象範囲**:
- システム全体アーキテクチャ
- Frontend設計（Angular）
- Backend設計（ASP.NET Core）
- データベース設計（SQL Database）
- セキュリティ設計
- パフォーマンス設計

**主要セクション**:
1. はじめに（目的、範囲、参照資料）
2. 設計概要（設計方針、アーキテクチャ、技術スタック）
3. システムコンテキスト
4. Frontend設計
   - コンポーネント構成
   - サービス設計
   - データモデル
   - ルーティング
5. Backend設計
   - API設計（全エンドポイント詳細）
   - Controller設計
   - Service Layer設計
   - Repository Layer設計
   - DTO設計
   - Validator設計
   - Middleware設計
6. データベース設計
   - ER図
   - テーブル定義
   - インデックス設計
   - Entity Framework Core設計
   - Migration Script
7. セキュリティ設計
   - OWASP Top 10対策
   - 入力値検証
   - APIレート制限
8. パフォーマンス設計
   - データベース最適化
   - キャッシング戦略
   - 非同期処理
9. トレーサビリティ（要件→設計マッピング）
10. 改訂履歴
11. 付録（命名規則、環境設定）

**対応要件**:
- 機能要件: REQ-FUNC-001～017 (全17件)
- 非機能要件: 全38件（REQ-PERF-001～008, REQ-USE-001～008, REQ-SEC-001～006, REQ-REL-001～005, REQ-MAIN-001～005, REQ-EXT-001～004, REQ-COMP-001～004）

**v2.0での主要変更**:
- Angular 17+最新機能完全対応（standalone、@if/@for、inject()、Signals）
- 非機能要件38件全てのトレーサビリティ追加
- 日本語全文検索の詳細設計（Japanese Word Breaker使用）
- 100万レコード対応の性能設計根拠を追加
- Service層の責務分離（TodoService、LabelService、SearchService）
- キーボードショートカット実装詳細
- 学習容易性設計（チュートリアル、ツールチップ）
- Angular Materialコンポーネント一覧
- 性能測定環境の明記

## レビュープロセス

1. **Draft作成**: 設計担当者が初版を作成
2. **Self Review**: 設計担当者による自己レビュー
3. **Peer Review**: チームメンバーによるレビュー
4. **Architect Review**: アーキテクトによる最終レビュー
5. **Approved**: 承認後、実装フェーズへ

## 参照方法

### 設計書の読み方

1. **初回読者（実装者）**:
   - まず「2. 設計概要」でアーキテクチャ全体を把握
   - 担当レイヤー（Frontend/Backend）のセクションを詳細に読む
   - 「9. トレーサビリティ」で実装する要件を確認

2. **レビュー担当者**:
   - 「9. トレーサビリティ」で要件カバレッジを確認
   - 各セクションの設計品質を品質基準に照らして確認
   - セキュリティ・パフォーマンス設計の妥当性を検証

3. **新規参加者**:
   - 「1. はじめに」と「2. 設計概要」を読む
   - 図表を中心に全体像を把握
   - 必要に応じて詳細セクションを参照
