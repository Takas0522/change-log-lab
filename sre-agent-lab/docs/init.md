# アプリケーションの構成について

- アプリケーションは簡易なToDo機能を提供します
- 独自の認証基盤を有しており、認証を通すことで自分のToDoを参照することが可能です
- WebAPIへの通信はJWTトークンの検証が成功する認証のみが許可されます
  - ログインフローは除く
- ユーザーは下記の項目を管理・参照できます
  - タイトル
  - 本文
  - ステータス（未着手/着手中/完了）
  - 作成日時（参照のみ）
  - 完了予定日時
  - 完了日（参照のみ）
- ユーザーは一覧からステータスの状態を変更することが可能ですが、そのたの項目を編集する場合は詳細画面に遷移する必要があります
- ユーザーは詳細画面からToDoを削除することが可能です
- ユーザーは一覧画面で記入されている項目をベースに絞り込みを行うことが可能です

# 技術要件

- Angular(latest)
  - CSSはtailwindを使用
- .NET(LTS latest)
- PostgreSQL

ソースコード類は `src` に格納されます。

# 開発要件

- まだ構築されていませんが、DevContainerを利用してローカルで開発を行います
- 最終的にはAzureで動かす想定ですがIaCを組む想定はありません。CI/CDパイプラインのみを構築する想定です
  - データが存在しない場合の初期データSeedの投入もCIに含まれます
  - パブリックに公開するアプリケーションであるためセキュリティに留意したデータとコードの構成を意識してください
- デモアプリケーションであるためテストの実装は必要ありません

# 開発の順序

1. 仕様の作成
  - 作成した仕様は`docs`に格納する
2. 開発計画の作成
  - 開発者が開発しやすい粒度で開発計画を作成し `docs/plans` に格納する
3. 開発
  - 開発計画に沿って開発
