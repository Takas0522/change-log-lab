# コードレビューチェックリスト

本チェックリストは、コードレビュー時に適用する詳細な確認項目を技術スタック別に定義する。

---

## 共通チェックリスト（すべてのコード）

### 正確性

- [ ] 仕様・要件に対するロジックの正しさを確認
- [ ] 境界値・エッジケースが適切に処理されている
- [ ] ガード条件（null チェック、範囲チェック）が必要な箇所に存在する
- [ ] オフバイワンエラーの有無を確認
- [ ] 戻り値の型と呼び出し元の期待が一致している

### 入力検証とセキュリティ

- [ ] 外部入力（HTTP リクエスト、クエリパラメータ、フォーム）が検証されている
- [ ] 認証・認可の漏れがない（`[Authorize]` 属性、ガード等）
- [ ] 秘匿情報（パスワード、API キー、接続文字列）がハードコードされていない
- [ ] SQL インジェクション / XSS / CSRF のリスクがない
- [ ] CORS 設定が適切（`AllowAnyOrigin` の不使用）

### エラーハンドリング / ロギング

- [ ] 例外の握り潰し（空 `catch`）がない
- [ ] 汎用 `catch (Exception)` の濫用がない
- [ ] 再スロー時に `throw;`（スタックトレース保持）を使用している
- [ ] 構造化ログ（プレースホルダー形式）を使用している
- [ ] グローバル例外ハンドリングの原則に従っている（各所での try/catch は原則不要）

### パフォーマンス

- [ ] 不要な同期 I/O がない
- [ ] 重複計算・冗長な処理がない
- [ ] N+1 クエリ問題が発生していない
- [ ] 不要なメモリ割り当て（大量データの `ToList()` 等）がない
- [ ] 適切なキャッシュ戦略が適用されている（該当箇所）

### 設計・保守性

- [ ] 命名が一貫性をもち、意図を表現している
- [ ] 関数・クラスが単一責任の原則（SRP）に従っている
- [ ] 循環依存がない
- [ ] マジックナンバー・マジックストリングが定数化されている
- [ ] 重複コード（DRY 原則違反）がない
- [ ] コメントと実装が一致している

### 並行性 / スレッド安全性

- [ ] 共有可変状態へのアクセスが保護されている
- [ ] 競合条件（race condition）のリスクがない
- [ ] デッドロックのリスクがない
- [ ] 非同期 API が正しく使用されている

### テスト

- [ ] ユニットテスト / 統合テストが存在する（ビジネスロジック層）
- [ ] クリティカルロジックのテストカバレッジが十分
- [ ] エッジケース・異常系のテストがある
- [ ] モックが適切に使用されている（外部依存のみモック）
- [ ] テスト名が意図を明確に表現している

### ドキュメント

- [ ] Public API・クラス・メソッドに XML ドキュメントコメント / JSDoc が記述されている
- [ ] 複雑なロジックに適切なインラインコメントがある
- [ ] README / API 仕様が変更に追従している

## C#（.NET / ASP.NET Core）チェックリスト

### 非同期の原則

- [ ] `async/await` を使用し、`.Result` / `.Wait()` を併用していない
- [ ] `CancellationToken` が適切に伝播されている
- [ ] `ConfigureAwait(false)` の使用が妥当（ライブラリコードのみ）
- [ ] 非同期メソッドの命名が `Async` サフィックスを持つ

### Nullable 参照型

- [ ] `#nullable enable` が設定されている
- [ ] null 安全の扱い（`?`, `!`）が妥当
- [ ] 未初期化 / ヌル参照のリスクがない

### 例外設計

- [ ] グローバルエラーハンドリングを使用し、各所での try/catch を避けている
- [ ] 空 `catch` ブロックがない
- [ ] 特定例外をハンドリングしている（汎用 Exception ではない）
- [ ] 例外メッセージが明確で、デバッグに有用
- [ ] ドメイン固有の例外クラスが適切に定義・使用されている

### DI とライフタイム

- [ ] `Singleton` / `Scoped` / `Transient` の設定が適切
- [ ] Scoped サービスを Singleton から解決していない
- [ ] Service Locator パターンを使用していない

### IDisposable

- [ ] `IDisposable` / `IAsyncDisposable` を実装するオブジェクトが `using` / `await using` で管理されている
- [ ] アンマネージリソースのリークがない

### LINQ / EF Core

- [ ] N+1 問題への対策（`Include` / `ThenInclude`）がある
- [ ] 読み取り専用クエリに `AsNoTracking()` が適用されている
- [ ] 不要な `ToList()` / `ToArray()` がない（遅延評価の活用）
- [ ] クエリがパラメータ化されている（`ExecuteSqlRaw` 使用時）
- [ ] トランザクション管理が適切

### 構造化ロギング

- [ ] `ILogger` のプレースホルダー形式（`LogInformation("User {Id} ...", id)`）を使用
- [ ] 文字列連結・補間によるログ出力がない
- [ ] 機密情報がログに含まれていない

### API / モデル

- [ ] DTO とエンティティが分離されている
- [ ] バリデーション属性（`[Required]`、`[StringLength]` 等）が適用されている
- [ ] `DateTime` vs `DateTimeOffset` の選択が適切
- [ ] タイムゾーンが考慮されている

## Angular（TypeScript / HTML / SCSS）チェックリスト

### TypeScript

- [ ] `strict` モードが有効
- [ ] `any` の使用がない（不確実な場合は `unknown`）
- [ ] 型推論が明らかな箇所で冗長な型注釈がない

### コンポーネント設計

- [ ] スタンドアロンコンポーネントを使用（NgModules 不使用）
- [ ] Angular v20+ では `standalone: true` を明示記述していない（デフォルト）
- [ ] `input()` / `output()` 関数を使用（`@Input` / `@Output` デコレータ不使用）
- [ ] `changeDetection: ChangeDetectionStrategy.OnPush` が設定されている
- [ ] `@HostBinding` / `@HostListener` を使用していない（`host` オブジェクトを使用）
- [ ] 静的画像に `NgOptimizedImage` を使用

### 状態管理

- [ ] ローカルコンポーネント状態にシグナルを使用
- [ ] 派生状態に `computed()` を使用
- [ ] `mutate` ではなく `update` / `set` を使用

### テンプレート

- [ ] ネイティブ制御フロー（`@if` / `@for` / `@switch`）を使用
- [ ] `*ngIf` / `*ngFor` / `*ngSwitch` を使用していない
- [ ] `ngClass` ではなく `class` バインディングを使用
- [ ] テンプレート内にグローバル参照（`new Date()` 等）がない
- [ ] テンプレート内にアロー関数がない

### フォーム

- [ ] リアクティブフォームを使用（テンプレート駆動フォーム不使用）

### RxJS

- [ ] `async` パイプを活用し、手動 `subscribe` を最小化
- [ ] 手動 `subscribe` の場合、`takeUntil` 等で確実に解放
- [ ] `Subject` の過剰使用がない

### サービス

- [ ] シングルトンサービスに `providedIn: 'root'` を使用
- [ ] `inject()` 関数を使用（コンストラクタインジェクション不使用）

### ルーティング

- [ ] 機能ルートに遅延ロードが実装されている
- [ ] ガード / リゾルバーが適切に設定されている

### セキュリティ

- [ ] `[innerHTML]` 使用時に `DomSanitizer` で制御
- [ ] HTTP インターセプタで認証ヘッダーが管理されている
- [ ] `bypassSecurityTrust*` の不適切な使用がない

### アクセシビリティ

- [ ] AXE チェックをパスする設計
- [ ] WCAG AA 最小要件（フォーカス管理、カラーコントラスト、ARIA 属性）に準拠
- [ ] フォームのエラーメッセージが適切

### テスト

- [ ] `TestBed` を用いたコンポーネントテストが存在
- [ ] 非同期テストの完了待ち（`fakeAsync` / `waitForAsync`）が正しい

## SQL チェックリスト

### 安全性

- [ ] パラメータ化クエリを使用（動的 SQL の文字列連結を禁止）
- [ ] 権限は最小限の原則に従っている
- [ ] シークレット（パスワード等）がコードに埋め込まれていない

### スキーマ整合性

- [ ] 主キー / 外部キー / 一意制約 / チェック制約が適用されている
- [ ] NULL / DEFAULT の設定が妥当
- [ ] マイグレーションが整備されている

### パフォーマンス

- [ ] 必要なインデックスが定義されている
- [ ] `SELECT *` を使用していない
- [ ] `EXPLAIN` / 実行計画のボトルネックを確認済み

### クエリ品質

- [ ] CTE / ウィンドウ関数が適切に使用されている
- [ ] 集約の正当性が確認されている
- [ ] N+1 を誘発する逐次クエリがない

### トランザクション

- [ ] 適切な分離レベルが設定されている
- [ ] コミット / ロールバックが確実に行われる
- [ ] 長期ロックやデッドロックのリスクがない
